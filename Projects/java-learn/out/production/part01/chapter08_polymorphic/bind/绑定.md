+ 绑定的概念
    + 在对象引用与对象实例之间建立联系，分为静态绑定和动态绑定
    
+ 静态绑定
    + 又称之为前期绑定，是在编译期建立的联系
    + 在编译时就建立的，性能较好，可以提高代码的运行速度
    + 类属性
    + 构造方法
    + 声明为 static 的静态方法
    + 声明为 fianl 的方法，将其另列为一个项目，如下声明为private的方法
    + 使用super调用的方法，包括super()
    
+ 动态绑定
    + 又称之为后期绑定，是在运行期建立的联系
    + 根据对象引用所指向的具体实例决定执行什么样的方法，由于在运行时由JVM完成的，因此会减慢程序运行速度
    + 提高程序灵活性，降低耦合度
